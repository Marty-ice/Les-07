Reflectie

Ik heb voor het keuzevak HTML/CSS gekozen omdat ik mijzelf een nieuwe handige tool wilde aanleren. Als CMD’er is het goed om te weten hoe websites geprogrammeerd worden. Tijdens de eerste les maakte ik voor de tweede keer kennis met de programmeer taal HTML. Ik had in het verleden al een cursus HTML programmeren voor een gedeelte gevolgd. Hierdoor wist ik al het één en ander af van de basis. HTML staat voor HyperText Markup Language en zorgt voor de structuur van een website. Als een website een menselijk lichaam is, dan stelt HTML de botten voor en CSS de huid. Ik heb geleerd tijdens het keuzevak dat je doormiddel van verschillende tags je website een structuur kan geven. Nadat ik de basis van HTML onder de knie had gekregen, mocht ik mijn HTML bestand opmaken met CSS. CSS is een andere programmeertaal waar mee je je HTML structuur een stijl kan geven. Door bijvoorbeeld het font te wijzigen, achtergrond kleuren geven of het positie geven van je tags. De les daarop heb ik kennis gemaakt met GitHub. Via GitHub kan je makkelijk je codes en andere software delen met andere gebruikers. Ook geeft GitHub je toegang tot eerdere versies van je code. De rest van de lessen bestonden uit opdrachten maken met betrekking tot HTML en CSS.

Actiepunten

De actiepunten die ik mijzelf mee geef wanneer ik weer ga programmeren gaan als volgt:

  1. Je ontwerpt eerst een schets (op papier of met Sketch/Adobe)
  2. Tijdens het ontwerp ga je nadenken hoe je je elementen gaat positioneren en welke tags je hierbij gaat gebruiken
  3. Een HTML bestand aanmaken met daarbij de structuur tags: !Doctype, html, header, body & footer
  4. Vervolgens voeg je je andere tags en elementen toe die je hebt ontworpen binnen je schets
  5. Wanneer je al je elementen hebt toegevoegd, maak je een een CSS bestand aan
  6. Binnen je CSS bestand maak je de tags en elementen op door ze te positioneren, grote te geven, kleuren etc.

Wat mij is opgevallen, is dat je doormiddel van zoeken naar de desbetreffende elementen, tags en attributen heel veel leert. Ik maak zelf veel gebruik van W3 schools of Modzilla Developer. Deze websites geven duidelijke en op een interactieve manier weer hoe je gebruik kan maken van verschillende elementen. Als beginnend programmeer moet je natuurlijk ergens beginnen en deze websites zorgen voor een goede basis. Verder is je design nooit af en zal je steeds opnieuw revisie rondes houden waarbij je je geschreven code opnieuw zal formuleren
